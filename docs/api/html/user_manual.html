iZprime User Manual

Scope

This manual summarizes the public API, toolkit internals, and extension conventions for the iZprime C library.

Architecture Layers

  • iZ Public API provides stable entry points used by applications.
  • iZ Toolkit covers iZ/iZm primitives and VX segment execution.
  • Bitmap Toolkit and Integer Arrays provide storage utilities used by sieves.
  • Utilities and Logging provide shared runtime helpers.

Algorithm Families

  • Classical sieves: SoE, SSoE, SoEu, SoS, SoA.
  • SiZ family: SiZ, SiZm, SiZm_vy.
  • Range variants: SiZ_stream and SiZ_count.
  • Prime search: vx_random_prime, vy_random_prime, iZ_next_prime.

Detailed pseudocode for the algorithms is available in docs/pseudocode.pdf.

Memory Ownership

  • Functions returning UI64_ARRAY * transfer ownership to the caller.
  • Release integer arrays with ui16_free, ui32_free, or ui64_free.
  • Release bitmaps with bitmap_free.
  • Release toolkit objects with iZm_free and vx_free.

Extending the Library

  1. Add API declarations and Doxygen contracts in headers under include/.
  2. Implement in src/ with sectioned comments that describe algorithm flow.
  3. Keep iZ mapping invariants explicit (6x-1 and 6x+1 lines).
  4. Add tests/benchmarks through the test API entry points.
