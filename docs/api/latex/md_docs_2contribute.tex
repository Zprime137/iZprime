Thanks for contributing. This project is prepared for research-\/grade publication, so we hold changes to a strict quality bar.\hypertarget{md_docs_2contribute_autotoc_md23}{}\doxysection{\texorpdfstring{Scope}{Scope}}\label{md_docs_2contribute_autotoc_md23}
Contributions are welcome for:\+


\begin{DoxyItemize}
\item correctness fixes,
\item performance improvements,
\item documentation quality,
\item tests/\+benchmarks and tooling,
\item new algorithms that align with the i\+Zprime design.
\end{DoxyItemize}\hypertarget{md_docs_2contribute_autotoc_md24}{}\doxysection{\texorpdfstring{Development Setup}{Development Setup}}\label{md_docs_2contribute_autotoc_md24}
Required dependencies:\+


\begin{DoxyItemize}
\item C compiler (GCC/\+\+Clang),
\item {\ttfamily make},
\item GMP,
\item Open\+SSL.
\end{DoxyItemize}

Optional:\+


\begin{DoxyItemize}
\item {\ttfamily pkg-\/config},
\item Doxygen,
\item Python packages in {\ttfamily py\+\_\+tools/\+requirements.\+txt} for plots.
\end{DoxyItemize}\hypertarget{md_docs_2contribute_autotoc_md25}{}\doxysection{\texorpdfstring{Code Style}{Code Style}}\label{md_docs_2contribute_autotoc_md25}

\begin{DoxyItemize}
\item Keep modules focused and composable.
\item Prefer explicit ownership and cleanup paths.
\item Keep public API declarations in {\ttfamily include/\+}, implementations in {\ttfamily src/\+}.
\item Document new public structs/\+functions with Doxygen comments.
\item Avoid introducing hidden global state.
\end{DoxyItemize}\hypertarget{md_docs_2contribute_autotoc_md26}{}\doxysection{\texorpdfstring{Validation Requirements}{Validation Requirements}}\label{md_docs_2contribute_autotoc_md26}
Before opening a pull request:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{make\ clean}
\DoxyCodeLine{make\ test-\/unit}
\DoxyCodeLine{make\ test-\/integration}
\DoxyCodeLine{make\ -\/-\/\ test-\/all\ -\/-\/verbose}

\end{DoxyCode}


If you touched performance-\/sensitive code, also run:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{make\ benchmark-\/p\_sieve\ save-\/results}
\DoxyCodeLine{make\ benchmark-\/p\_gen\ save-\/results}
\DoxyCodeLine{make\ benchmark-\/SiZ\_count\ save-\/results}

\end{DoxyCode}


If you changed API/\+docs:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{make\ userManual}

\end{DoxyCode}
\hypertarget{md_docs_2contribute_autotoc_md27}{}\doxysection{\texorpdfstring{Benchmarking Expectations}{Benchmarking Expectations}}\label{md_docs_2contribute_autotoc_md27}

\begin{DoxyItemize}
\item Include the exact command(s) used.
\item Include hardware/\+\+OS/\+compiler context.
\item Share result files from {\ttfamily output/\+} when relevant.
\item Prefer reporting both absolute timing and relative speedup/\+regression.
\end{DoxyItemize}\hypertarget{md_docs_2contribute_autotoc_md28}{}\doxysection{\texorpdfstring{Pull Request Checklist}{Pull Request Checklist}}\label{md_docs_2contribute_autotoc_md28}

\begin{DoxyEnumerate}
\item Explain the problem and the technical approach.
\item List behavioral changes and compatibility impact.
\item Link tests that validate the change.
\item Add or update benchmark evidence for performance claims.
\item Update docs ({\ttfamily README.\+md}, {\ttfamily docs/\+\texorpdfstring{$\ast$}{*}.md}, Doxygen comments) when needed.
\end{DoxyEnumerate}\hypertarget{md_docs_2contribute_autotoc_md29}{}\doxysection{\texorpdfstring{Algorithm Contributions}{Algorithm Contributions}}\label{md_docs_2contribute_autotoc_md29}
For new sieve/\+generation techniques:\+


\begin{DoxyItemize}
\item map design to existing toolkit abstractions ({\ttfamily \doxylink{struct_i_z_m}{IZM}}, {\ttfamily \doxylink{struct_v_x___s_e_g}{VX\+\_\+\+SEG}}, bitmaps, arrays),
\item provide pseudocode-\/level explanation in {\ttfamily docs/\+pseudocode.\+pdf} source pipeline,
\item add integration tests versus trusted baselines,
\item add benchmark hooks so users can compare against existing models.
\end{DoxyItemize}\hypertarget{md_docs_2contribute_autotoc_md30}{}\doxysection{\texorpdfstring{Review Standard}{Review Standard}}\label{md_docs_2contribute_autotoc_md30}
PRs are reviewed for:\+


\begin{DoxyItemize}
\item correctness and edge-\/case behavior,
\item memory/\+process safety,
\item API clarity and consistency,
\item test coverage,
\item reproducibility of benchmark claims. 
\end{DoxyItemize}
