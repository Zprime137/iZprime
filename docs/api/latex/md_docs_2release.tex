This document defines the release workflow for {\ttfamily v1.\+x} and later.\hypertarget{md_docs_2release_autotoc_md50}{}\doxysection{\texorpdfstring{Versioning}{Versioning}}\label{md_docs_2release_autotoc_md50}

\begin{DoxyItemize}
\item Use Sem\+Ver tags:\+ {\ttfamily v\+MAJOR.\+MINOR.\+PATCH} (example:\+ {\ttfamily v1.\+0.\+0}).
\item {\ttfamily VERSION} in {\ttfamily Makefile} should match the release tag.
\item {\ttfamily SOVERSION} defaults to the major version and should change only on ABI breaks.
\end{DoxyItemize}\hypertarget{md_docs_2release_autotoc_md51}{}\doxysection{\texorpdfstring{Pre-\/release Gate}{Pre-\/release Gate}}\label{md_docs_2release_autotoc_md51}
Run this checklist from a clean working tree:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{make\ clean}
\DoxyCodeLine{make\ test-\/unit}
\DoxyCodeLine{make\ test-\/integration}
\DoxyCodeLine{make\ benchmark-\/p\_sieve\ save-\/results}
\DoxyCodeLine{make\ benchmark-\/p\_gen\ save-\/results}
\DoxyCodeLine{make\ benchmark-\/SiZ\_count\ save-\/results}
\DoxyCodeLine{make\ userManual}
\DoxyCodeLine{make\ dist\ VERSION=1.0.0}

\end{DoxyCode}


Confirm:\+


\begin{DoxyItemize}
\item generated {\ttfamily docs/\+user\+Manual.\+pdf} opens correctly,
\item benchmark outputs are saved in {\ttfamily output/\+},
\item {\ttfamily dist/\+} contains tarball + checksum,
\item release tarball does not include transient artifacts ({\ttfamily build/\+}, {\ttfamily logs/\+}, {\ttfamily output/\+}, {\ttfamily .git/\+}).
\end{DoxyItemize}\hypertarget{md_docs_2release_autotoc_md52}{}\doxysection{\texorpdfstring{Changelog and Release Notes}{Changelog and Release Notes}}\label{md_docs_2release_autotoc_md52}
{\ttfamily Change\+Log} should summarize user-\/visible changes.

Suggested generation workflow:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{LAST\_TAG=\$(git\ describe\ -\/-\/tags\ -\/-\/abbrev=0\ 2>/dev/null\ ||\ echo\ "{}"{})}
\DoxyCodeLine{if\ [\ -\/n\ "{}\$LAST\_TAG"{}\ ];\ then}
\DoxyCodeLine{\ \ git\ log\ -\/-\/pretty=format:"{}-\/\ \%s\ (\%h)"{}\ "{}\$LAST\_TAG"{}..HEAD}
\DoxyCodeLine{else}
\DoxyCodeLine{\ \ git\ log\ -\/-\/pretty=format:"{}-\/\ \%s\ (\%h)"{}}
\DoxyCodeLine{fi}

\end{DoxyCode}


Use the output as a draft, then curate into sections:\+


\begin{DoxyItemize}
\item Added
\item Changed
\item Fixed
\item Performance
\item Documentation
\end{DoxyItemize}

Release notes in Git\+Hub/\+\+Git\+Lab should include:\+


\begin{DoxyItemize}
\item highlights,
\item migration notes (if any),
\item validation commands used,
\item benchmark artifacts summary.
\end{DoxyItemize}\hypertarget{md_docs_2release_autotoc_md53}{}\doxysection{\texorpdfstring{Tag and Publish}{Tag and Publish}}\label{md_docs_2release_autotoc_md53}

Attach {\ttfamily dist/\+izprime-\/\texorpdfstring{$<$}{<}version\texorpdfstring{$>$}{>}.tar.\+gz} and checksum file to the release.\hypertarget{md_docs_2release_autotoc_md54}{}\doxysection{\texorpdfstring{Post-\/release}{Post-\/release}}\label{md_docs_2release_autotoc_md54}

\item track follow-\/up fixes as issues/\+milestone items.
